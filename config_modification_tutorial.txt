crispz-studio - config tutorial
================================

HOW IT WORKS
------------
Settings live in a JSON file loaded at startup. Priority:
  1) config.txt        -> your LOCAL copy (gitignored, never pushed). Edit this one.
  2) config-sample.txt -> the generic template shipped in the repo (fallback).
  3) built-in defaults -> used if a key (or both files) is missing.

First time: copy config-sample.txt to config.txt, then edit config.txt.
  Windows : copy config-sample.txt config.txt
  Linux   : cp config-sample.txt config.txt
Restart the app after editing. (config.txt is NOT version-controlled, so your
local paths/tweaks stay private.)


KEYS
----
ollama_url              Ollama server URL (Describe + Improve prompt).
                        e.g. "http://localhost:11434"
ollama_describe_prompt  Instruction sent to the vision model to caption an image
                        into a prompt (Describe tab).
ollama_improve_prompt   Instruction to rewrite/enrich the current prompt. Must
                        contain {prompt} (replaced by your prompt). Edit the tone
                        here (e.g. ask for photographic detail, a language, etc.).
ollama_compose_prompt   Instruction for the Vision Mix tab: merges the per-image
                        descriptions into ONE prompt. Should contain {descriptions}
                        (replaced by the list of captions).
ollama_vision_blocklist List of model-name substrings to EXCLUDE from the vision
                        model list (Describe / Vision Mix), even if Ollama reports
                        them as vision-capable. e.g. ["qwen3.6"] to hide a model
                        that claims vision but doesn't work well.
ollama_cpu              true -> run the Ollama vision/LLM on CPU (num_gpu=0): zero
                        VRAM shared with Z-Image (slower captions). Good if your GPU
                        is tight when using Describe / Vision Mix during generation.
ollama_keep_alive       How long Ollama keeps the model in VRAM after a call.
                        0 (default) = unload immediately so the VRAM is freed
                        before Z-Image generates (avoids GPU competition on one
                        card). Use "30s"/"5m" or -1 (keep) if you have VRAM to
                        spare and want faster repeated Describe/Vision Mix.

default_prompt          Prompt prefilled at startup.
default_negative_prompt Negative prompt prefilled at startup.
default_styles          List of style names checked by default, e.g.
                        ["Fooocus Cinematic", "SAI Anime"].
default_aspect_ratio    One of the Settings aspect labels, e.g. "1024 x 1024  (1:1)".
default_performance     One of the performance_presets names below.
performance_presets     Editable Performance presets: name -> [steps, guidance].
                        Add/rename your own (e.g. "Juggernaut": [28, 6.0]).
model_profiles          Per-model auto-tuning: a substring of the model name ->
                        {steps, guidance}. When you pick a checkpoint / set a
                        transformer override, steps + CFG auto-adjust to the first
                        matching profile (e.g. "juggernaut" -> CFG 6). 0 guidance =
                        Turbo-style; >0 = Base/CFG models.
default_model_profile   {steps, guidance} used when no model_profiles key matches.
default_width/height    txt2img size (multiples of 16).
default_gen_steps       txt2img steps.
default_guidance        CFG. 0 = Z-Image Turbo. Z-Image Base: ~3.5-5.
default_sampler         Scheduler par defaut: euler (natif flow), dpmpp2m (DPM++ 2M),
                        dpm2a (ancestral, experimental). Choix aussi dans l'UI (a cote du CFG).
default_image_number    Batch size at startup (1-30).
default_seed            -1 = random.

default_factor          Upscale factor (img2img/upscale).
default_denoise         Refine denoise strength (0..0.8).
default_refine          Default state of the "Refine (img2img)" checkbox (true/false).
                        false = ESRGAN upscale only by default (skip the slow refine).
default_refine_first    Default state of "Refine before upscale (faster)" (true/false).
                        true = refine at native res THEN ESRGAN (~4-16x faster refine).
default_refine_steps    Z-Image refine steps.
default_tile/overlap    ESRGAN tiling (0 = off) to cap VRAM.
default_refine_tile/overlap   Z-Image diffusion tiling for 4K+ (0 = whole image).
default_cpu_offload     "none" | "model" (~half VRAM) | "sequential" (~9GB, slower).

checkpoints_dir         Folder of single-file Z-Image models (.safetensors). Scanned
                        into the Models > Checkpoints dropdown to switch model (like
                        ESRGAN). Empty -> ./checkpoints next to app.py.
loras_dir               Folder of LoRA .safetensors. Scanned into Models > LoRA.
                        Empty -> ./loras next to app.py.
wildcards_dir           Folder of wildcard .txt files. In the prompt, __name__ is
                        replaced by a random line from <folder>/name.txt (e.g.
                        __color__). Nested wildcards supported, reproducible per
                        seed. Empty -> ./wildcards next to app.py.
default_lora_weight     Default LoRA strength (0..2).
zimage_omni_model       HF repo id OR local diffusers folder of a Z-Image Omni/Edit
                        model (with a SigLIP encoder). REQUIRED for the Reference
                        (Omni) multi-image mode. The Turbo/Base text-to-image models
                        cannot do Omni (no SigLIP).
                        NOTE: as of now Tongyi has released only Z-Image-Turbo and
                        Z-Image-Base. 'Z-Image-Omni-Base' and 'Z-Image-Edit' are still
                        "coming soon". Leave this empty until they ship, then set it
                        (likely "Tongyi-MAI/Z-Image-Omni-Base" or "Tongyi-MAI/Z-Image-Edit").
faceswap_model_path     Absolute path to an inswapper .onnx file (Face Swap tab). If
                        empty, the app also looks for inswapper_128.onnx in the
                        'faceswap' folder (next to app.py), 'models', checkpoints_dir,
                        and ~/.insightface/models. Needs: pip install insightface onnxruntime-gpu.
faceswap_model_url      Optional URL to auto-download inswapper_128.onnx once (into the
                        'faceswap' folder) if the model is not found locally.

default_save_mode       "display" (no save) | "local" | "alongside" | "custom".
default_output_dir      Folder for "local"/"custom" save modes.
default_output_format   "png" | "jpg" | "webp".
date_subfolders         true (default) -> save into out/YYYY-MM-DD/ subfolders
                        (like Fooocus). The galleries/Asset Browser scan recursively.
filename_pattern        Output filename (no extension). Placeholders:
                        {date}  YYYYMMDD-HHMMSS
                        {tag}   upscaled / txt2img / img2img / omni / faceswap (+ source name)
                        {seed}  the seed used (or "rand")
                        {w} {h} output size
                        {index} _1.._N for batches
                        Files are made unique (_2, _3...) so nothing is overwritten.

log_level               "quiet" | "info" | "debug". debug = full params/state/VRAM
                        in the console (also: run.bat --debug, or --log-level debug).
