crispz-studio - config tutorial
================================

HOW IT WORKS
------------
Settings live in a JSON file loaded at startup. Priority:
  1) config.txt        -> your LOCAL copy (gitignored, never pushed). Edit this one.
  2) config-sample.txt -> the generic template shipped in the repo (fallback).
  3) built-in defaults -> used if a key (or both files) is missing.

First time: copy config-sample.txt to config.txt, then edit config.txt.
  Windows : copy config-sample.txt config.txt
  Linux   : cp config-sample.txt config.txt
Restart the app after editing. (config.txt is NOT version-controlled, so your
local paths/tweaks stay private.)


KEYS
----
ollama_url              Ollama server URL (Describe + Improve prompt).
                        e.g. "http://localhost:11434"
ollama_describe_prompt  Instruction sent to the vision model to caption an image
                        into a prompt (Describe tab).
ollama_improve_prompt   Instruction to rewrite/enrich the current prompt. Must
                        contain {prompt} (replaced by your prompt). Edit the tone
                        here (e.g. ask for photographic detail, a language, etc.).
ollama_compose_prompt   Instruction for the Vision Mix tab: merges the per-image
                        descriptions into ONE prompt. Should contain {descriptions}
                        (replaced by the list of captions).
ollama_vision_blocklist List of model-name substrings to EXCLUDE from the vision
                        model list (Describe / Vision Mix), even if Ollama reports
                        them as vision-capable. e.g. ["qwen3.6"] to hide a model
                        that claims vision but doesn't work well.
ollama_cpu              true -> run the Ollama vision/LLM on CPU (num_gpu=0): zero
                        VRAM shared with Z-Image (slower captions). Good if your GPU
                        is tight when using Describe / Vision Mix during generation.
ollama_keep_alive       How long Ollama keeps the model in VRAM after a call.
                        0 (default) = unload immediately so the VRAM is freed
                        before Z-Image generates (avoids GPU competition on one
                        card). Use "30s"/"5m" or -1 (keep) if you have VRAM to
                        spare and want faster repeated Describe/Vision Mix.

default_prompt          Prompt prefilled at startup.
default_negative_prompt Negative prompt prefilled at startup.
default_styles          List of style names checked by default, e.g.
                        ["Fooocus Cinematic", "SAI Anime"].
default_aspect_ratio    One of the Settings aspect labels, e.g. "1024 x 1024  (1:1)".
default_performance     One of the performance_presets names below.
performance_presets     Editable Performance presets: name -> [steps, guidance].
                        Add/rename your own (e.g. "Juggernaut": [28, 6.0]).
model_profiles          Per-model auto-tuning: a substring of the model name ->
                        {steps, guidance}. When you pick a checkpoint / set a
                        transformer override, steps + CFG auto-adjust to the first
                        matching profile (e.g. "juggernaut" -> CFG 6). 0 guidance =
                        Turbo-style; >0 = Base/CFG models.
default_model_profile   {steps, guidance} used when no model_profiles key matches.
default_width/height    txt2img size (multiples of 16).
default_gen_steps       txt2img steps.
default_guidance        CFG. 0 = Z-Image Turbo. Z-Image Base: ~3.5-5.
default_sampler         Sampler par defaut: euler (natif flow-matching) ou unipc
                        (UniPC multistep). Z-Image impose des sigmas custom -> DPM++/DPM2a
                        incompatibles. Choix aussi dans l'UI (a cote du CFG).
default_schedule        Schedule de sigmas (facon ComfyUI): sgm_uniform (natif Z-Image),
                        beta, karras, exponential. Choix aussi dans l'UI (a cote du sampler).
default_image_number    Batch size at startup (1-30).
default_seed            -1 = random.

default_factor          Upscale factor (img2img/upscale).
default_denoise         Refine denoise strength (0..0.8).
default_refine          Default state of the "Refine (img2img)" checkbox (true/false).
                        false = ESRGAN upscale only by default (skip the slow refine).
default_refine_first    Default state of "Refine before upscale (faster)" (true/false).
                        true = refine at native res THEN ESRGAN (~4-16x faster refine).
default_refine_steps    Z-Image refine steps.
default_tile/overlap    ESRGAN tiling (0 = off) to cap VRAM.
default_refine_tile/overlap   Z-Image diffusion tiling for 4K+ (0 = whole image).
attention_slice_above   Cote (px) au-dela duquel on active l'attention slicing (whole-image
                        2K+ -> evite le spill VRAM). En-dessous (tuiles 1024, txt2img) le
                        slicing est OFF = attention SDPA rapide. 1664 par defaut. Sur grosse
                        carte tu peux monter (ex. 2048) pour ne jamais slicer.
auto_refine_tile_above  Cote (px) au-dela duquel un refine "whole image" (refine_tile=0) est
                        auto-tuile (tuile 1024). Defaut = attention_slice_above (1664): au-dela
                        un whole-image serait slice (lent ~120s en 2K) et risque le spill 4K.
refine_tile_denoise_cap Plafond de denoise pour le refine TUILE (4K+). En tuiles, chaque tuile
                        est rediffusee avec le prompt global -> a fort denoise (Creative) la
                        diffusion RECREE le sujet (ex: la tasse) DANS chaque tuile = duplications.
                        On plafonne donc le denoise par tuile (defaut 0.40) pour que le contenu
                        existant guide la diffusion (facon Ultimate SD Upscale). 0 = pas de
                        plafond. Le refine "whole image" (image entiere) garde le denoise demande
                        (une seule passe -> aucune duplication possible). Baisse a 0.30 si tu vois
                        encore le sujet se repeter dans les tuiles.
refine_tile_prompt      Prompt utilise PAR TUILE pour le refine tuile. Le prompt global decrit
                        TOUTE la composition (pas la tuile) -> le passer a chaque tuile pousse la
                        diffusion a recreer le sujet (ex: la tasse) dans des tuiles de fond.
                        Defaut "" = prompt VIDE par tuile (chaque tuile affine juste le detail
                        local = pas de duplication). "global" ou "scene" = reutilise le prompt de
                        la scene (ancien comportement). Tout autre texte = prompt generique
                        applique a chaque tuile, ex: "high detail, sharp focus, fine texture".
                        Avec un prompt vide tu peux remonter refine_tile_denoise_cap sans
                        reintroduire de duplication.
default_cpu_offload     "none" | "model" (~half VRAM) | "sequential" (~9GB, slower).

checkpoints_dir         Folder of single-file Z-Image models (.safetensors). Scanned
                        into the Models > Checkpoints dropdown to switch model (like
                        ESRGAN). Empty -> ./checkpoints next to app.py.
loras_dir               Folder of LoRA .safetensors. Scanned into Models > LoRA.
                        Empty -> ./loras next to app.py.
wildcards_dir           Folder of wildcard .txt files. In the prompt, __name__ is
                        replaced by a random line from <folder>/name.txt (e.g.
                        __color__). Nested wildcards supported, reproducible per
                        seed. Empty -> ./wildcards next to app.py.
default_lora_weight     Default LoRA strength a slot starts at (default 1.0). Clamped
                        into lora_weight_min..lora_weight_max.
lora_weight_min         Min/max of the LoRA "Weight" sliders (default -2.0 / 2.0).
lora_weight_max         NEGATIVE weights are allowed and useful: the LoRA then pushes
                        AWAY from what it was trained on (a "skinny slider" at -1 gives
                        the opposite effect). 0 = no effect. Past about +/-1.5 most LoRAs
                        burn or deform the image. Set lora_weight_min to 0 to forbid
                        negatives. Bad values or min >= max -> falls back to -2..2.
lora_slots              Number of LoRA slots shown in Models > LoRA (1..10, default 3).
                        Each slot = a dropdown + weight; all combine at generation.
zimage_omni_model       HF repo id OR local diffusers folder of a Z-Image Omni/Edit
                        model (with a SigLIP encoder). REQUIRED for the Reference
                        (Omni) multi-image mode. The Turbo/Base text-to-image models
                        cannot do Omni (no SigLIP).
                        NOTE: as of now Tongyi has released only Z-Image-Turbo and
                        Z-Image-Base. 'Z-Image-Omni-Base' and 'Z-Image-Edit' are still
                        "coming soon". Leave this empty until they ship, then set it
                        (likely "Tongyi-MAI/Z-Image-Omni-Base" or "Tongyi-MAI/Z-Image-Edit").
faceswap_model_path     Absolute path to an inswapper .onnx file (Face Swap tab). If
                        empty, the app also looks for inswapper_128.onnx in the
                        'faceswap' folder (next to app.py), 'models', checkpoints_dir,
                        and ~/.insightface/models. Needs: pip install insightface onnxruntime-gpu.
faceswap_model_url      Optional URL to auto-download inswapper_128.onnx once (into the
                        'faceswap' folder) if the model is not found locally.

default_save_mode       "display" (no save) | "local" | "alongside" | "custom".
default_output_dir      Folder for "local"/"custom" save modes. Absolute paths are
                        allowed (e.g. another drive). WINDOWS PATHS IN JSON: use "/" or
                        "\\" -- a single "\" is an illegal JSON escape and makes the WHOLE
                        config.txt fail to load silently (every setting falls back to
                        config-sample.txt). Right: "E:/images/out" or
                        "E:\\images\\out". Wrong: "E:\images\out".
                        Changing the "Output folder" field in the UI at runtime is fine
                        too: the folder is auto-authorised for the Asset Browser (no more
                        "File not allowed"), no restart needed.
default_output_format   "png" | "jpg" | "webp".
civitai_api_key         Optional CivitAI API token. In the Asset Browser (LoRAs/Models
                        tabs), "Fetch from CivitAI" looks a model up by its SHA256, then
                        saves <name>.preview.png + <name>.civitai.json (trigger words +
                        example images). A key unlocks gated/NSFW previews; most public
                        models work without one.
date_subfolders         true (default) -> save into out/YYYY-MM-DD/ subfolders
                        (like Fooocus). The galleries/Asset Browser scan recursively.
save_pre_upscale        false (default). true -> in txt2img + auto-upscale, ALSO save the
                        original txt2img image (before ESRGAN/refine), tagged 'txt2img',
                        next to the final upscaled one. Toggle in Advanced > Generation.
wildcards_in_order      false (default) -> __name__ picks a RANDOM line (reproducible per
                        seed). true -> in a batch, image i takes line (i % nb_lines) of the
                        wildcard file (deterministic sweep). Toggle in Advanced > Generation.
metadata_scheme         "crispz" (default) | "a1111". a1111 also writes an A1111/Civitai
                        'parameters' text chunk into PNGs so Civitai reads the prompt +
                        seed + params. crispz's own chunk + sidecar .json are kept either
                        way. Switchable live in Advanced > Metadata.
filename_pattern        Output filename (no extension). Placeholders:
                        {date}  YYYYMMDD-HHMMSS
                        {tag}   upscaled / txt2img / img2img / omni / faceswap (+ source name)
                        {seed}  the seed used (or "rand")
                        {w} {h} output size
                        {index} _1.._N for batches
                        Files are made unique (_2, _3...) so nothing is overwritten.

log_level               "quiet" | "info" | "debug". debug = full params/state/VRAM
                        in the console (also: run.bat --debug, or --log-level debug).

hf_token                Hugging Face token for gated models (empty = anonymous).
caption_model           Local auto-describe model (no Ollama): "blip-large"
                        (default, richer) | "blip-base" (lighter). Used by the
                        Describe fallback and Inpaint/Outpaint Auto-describe.
                        Also selectable in Prompt AI > Caption model (persisted).
improve_local_keywords  Quality tags appended by "Improve prompt" when no Ollama
                        model is selected (local rule-based fallback).
checkpoints_extra_dir   Optional second checkpoints folder, merged into the single
                        "Z-Image checkpoint" dropdown ("" = disabled).

job_queue               {"enabled": true}
                        Job queue panel: "+ Queue" snapshots ALL current settings
                        (incl. model/LoRAs/sampler); "Run queue" chains the jobs;
                        Stop pauses (remaining jobs kept). enabled=false removes
                        the panel entirely (no components, zero cost).
xyz_grid                {"enabled": true, "max_jobs": 100, "thumb": 512,
                         "suggest": true}
                        X/Y/Z comparison grid (needs job_queue). Every combo
                        becomes a queued job; an annotated contact sheet is
                        assembled per Z value (X columns x Y rows, thumb px cells)
                        in <output>/xyz_<timestamp>/. max_jobs caps the combo
                        count. suggest=false disables the contextual value
                        suggestions (placeholder + fill button).
tag_autocomplete        {"enabled": true, "max_results": 8, "sources": [urls]}
                        Prompt/negative autocomplete. CSVs in sources are
                        downloaded once into tags/ (any .csv dropped there is
                        also used: "name,category,count,aliases" or one word per
                        line), merged with your local __wildcards__. Dropdown
                        under the caret: arrows + Tab/Enter, Escape. enabled=false
                        downloads nothing and injects no script (zero cost).
